GCD Module Verification

SiliconJackets March 2026

Designed a complete UVM (Universal Verification Methodology) testbench in SystemVerilog to verify a GCD (Greatest Common Divisor) RTL module. The verification effort uncovered multiple RTL bugs and achieved 99% code coverage using Cadence Xcelium.

Details
  • Implemented UVM agent components (driver, monitor, and sequencer) for both the GCD and Reset interfaces, instantiated within a structured UVM environment containing a golden-model scoreboard for result comparison.
  • Wrote constrained random stimulus sequences and directed test cases by extending UVM base sequence and test classes, targeting corner-case GCD behavior such as equal operands, zero inputs, and large co-prime pairs.
  • Simulated the design using Cadence Xcelium, achieving 99% functional code coverage and identifying and resolving multiple RTL bugs including incorrect reset behavior and off-by-one errors in the iterative subtraction logic.
